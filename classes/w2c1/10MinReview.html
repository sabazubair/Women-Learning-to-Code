HTML + Accessibility

  • Tech consumers vs Tech creators
  • "meaningless HTML elements" - divs, spans. Use sparingly.
  • Semantic* HTML - human and machine-readable ✅
  • HTML supports accessibility out of the box.
  • Web Content Accessibility Guidelines (WCAG) conducted an annual accessibility analysis of the top 1,000,000 home pages and found these common accessibility mistakes.
  1. low contrast - using bright colors on a white background, pastel colors etc. requires designers + frontend engineers to solve.
  2. missing alt-text - fix by providing a text explanation of your image.
  3. empty links - example: - provide hidden descriptive text. Certain sites keep descriptions hidden from view with class names like "sr-only". You can also use aria-labels.
  4. missing form labels - add form labels!
  5. empty buttons - use aria-labels or use hidden text styled as class ".sr-only" (screenreader-only).
  6. missing document language - easy fix! Why: "The lang attribute allows assistive technology to orient and adapt to the pronunciation and syntax that are specific to the language of the page."
