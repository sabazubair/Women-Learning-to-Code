Intro to Objects

  1. In our review, we saw that arrays are officially documented as "list-like objects". Remember that definition. You'll understand it entirely by the end of this lecture.
  2. Objects group together a set of variables and functions to create a model of something that you would recognize from the real world.
  3. In an object, variables become known as properties.
  4. In an object, functions become known as methods.
  5. Visually: objects have {} (in the same way arrays have []) but unlike arrays, objects are not indexed. Order does not matter. Objects are created in key/value pairs.
  6. Key's are unique!
  7. We can access data stored in an object using either the dot notation or the square bracket notation.
